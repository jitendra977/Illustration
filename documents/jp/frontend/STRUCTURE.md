# フロントエンドディレクトリ構造 (`/frontend`)

フロントエンドは、**Vite** と **Bun/NPM** で構築された **React SPA** であり、デザインシステムとして **Material UI (MUI)** を使用しています。

---

## 🌳 ファイルツリー
```text
frontend/
├── public/             # 静的公開アセット（ファビコン、ビデオ）
├── src/                # アプリケーションソースコード
│   ├── api/            # ベースとなる Axios 構成
│   ├── assets/         # グローバルスタイル、フォント、画像
│   ├── components/     # 再利用可能な UI ウィジェット
│   │   ├── admin/      # 特殊な管理パネル（ログ監査、ユーザー管理）
│   │   ├── common/     # 汎用ボタン、ローダーなど
│   │   └── ...
│   ├── context/        # グローバル状態プロバイダー（認証、テーマ）
│   ├── hooks/          # カスタム React フック（useAuth, useResponsive）
│   ├── layouts/        # ページシェル（モバイル、標準）
│   ├── pages/          # トップレベルのルートコンポーネント（ダッシュボード、ログイン）
│   ├── routes/         # 統一されたルーティング定義
│   ├── services/       # 機能固有の API クライアント
│   ├── theme/          # カスタム Material UI テーマ定義
│   ├── utils/          # ヘルパー（フォーマッタ、バリデータ）
│   ├── App.jsx         # メインアプリケーションエントリ
│   └── main.jsx        # ブラウザマウントポイント
├── Dockerfile          # マルチステージ本番環境コンテナ（Nginx）
├── Dockerfile.dev      # 高速リロード対応の開発用コンテナ
├── nginx.conf          # SPA サポート用 Nginx ルーティングルール
├── package.json        # プロジェクトメタデータと依存関係
└── vite.config.js      # Vite ビルドツールセットアップ
```

---

## 📦 主要コンポーネント

### 1. レスポンシブレイアウト (`/src/layouts`)
- **`MobileLayout.jsx`**: モバイルユーザー向けの常駐ボトムナビゲーションとスワイプ可能なドロワーを提供します。

### 2. 管理システム (`/src/components/admin`)
- **監査**: バックエンドからの `ActivityLogs` を表示するために特別に構築されたタブ。
- **ユーザー管理**: 複雑なユーザー/ロール関係を管理するためのモジュール化されたコンポーネント。

### 3. テーマシステム (`/src/theme`)
- **ブルースタイリッシュデザイン**: カスタム HSL ベースのカラーと完全なダークモード互換性を備えたプレミアムデザインシステム。
